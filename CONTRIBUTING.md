# Contributing

## 目标

本文定义本知识库的写作和维护规则，保证新增通信协议文档结构一致、内容可验证、便于长期扩展。

## 文档类型

本知识库主要包含四类文档：

1. 基础文档：说明协议或接口的定位、概念和硬件连接。
2. Practical 文档：面向工程调试，强调接线、参数、流程和故障定位。
3. Deep Dive 文档：深入机制、边界条件、驱动模型和复杂故障。
4. Reference 文档：选型、术语、工具、场景、对比和实验路线。

## 文件命名

推荐命名：

```text
basic/<name>.md
basic/<name>-practical.md
basic/<name>-deep-dive.md
bus/<name>.md
bus/<name>-practical.md
bus/<name>-deep-dive.md
```

组合主题使用短横线：

```text
rs485-modbus-rtu-practical.md
can-canopen-deep-dive.md
mipi-csi-dsi-debug.md
```

## 内容要求

每篇协议文档至少回答：

- 它解决什么问题？
- 用在哪些典型设备或系统？
- 硬件需要哪些信号和注意事项？
- 软件初始化或通信流程是什么？
- 最常见错误有哪些？
- 应该用什么工具调试？

## 风格要求

- 优先使用简短段落。
- 多用表格总结现象和排查方向。
- 避免只讲概念，不给工程检查项。
- 不确定的速率、限制和规范细节要注明“以手册为准”。
- 区分物理层、链路层、协议层和应用层。

## 新增文档检查清单

- 文件名符合命名规则。
- 已加入对应 README 索引。
- 有“定位”章节。
- 有“常见错误”或“故障定位”章节。
- 有检查清单或调试步骤。
- 有延伸阅读链接。
- 没有把不同层级概念混为一谈。

## 推荐流程

1. 先添加基础入口文档。
2. 再补 Practical Guide。
3. 最后补 Deep Dive。
4. 更新 `README.md`、分类 README 和相关 Reference 文档。
5. 提交时使用清晰提交信息。
