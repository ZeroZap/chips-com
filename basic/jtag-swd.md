# JTAG / SWD

## 定位

JTAG 和 SWD 是常见芯片调试接口，用于下载程序、断点调试、寄存器访问和边界扫描。

它们不是普通数据通信接口，但在芯片开发中非常重要。

## JTAG

常见信号：

```text
TCK
TMS
TDI
TDO
TRST，可选
GND
VTref
```

特点：

- 引脚较多。
- 支持链式连接多个器件。
- 可用于边界扫描。

## SWD

SWD 常见于 ARM MCU。

常见信号：

```text
SWCLK
SWDIO
NRST，可选
GND
VTref
```

特点：

- 引脚少。
- 适合 MCU 调试。
- 常用于量产烧录和在线调试。

## VTref

调试器通常需要目标板参考电压：

```text
VTref
```
它告诉调试器目标 IO 电平是 1.8V、3.3V 还是其他电压。

## 常见错误

| 现象 | 优先检查 |
| --- | --- |
| 连接不上 | 供电、GND、VTref、复位、引脚复用 |
| 下载中断 | 时钟过高、线长、接触不良 |
| 芯片锁死 | 低功耗、读保护、复位脚、BOOT 配置 |
| SWDIO 异常 | 上拉、复用、被应用占用 |

## 检查清单

- 调试器和目标板共地。
- VTref 正确连接。
- SWD/JTAG 引脚未被错误复用。
- NRST 可控。
- 下载时钟先降速验证。
- 量产烧录考虑测试点可靠性。
