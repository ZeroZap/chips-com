# RS232 / RS485

## 定位

`RS232` 和 `RS485` 是常见串口电气标准，经常承载 UART 字节流，但它们不是 UART 本身。

分层关系：

```text
UART：字节帧格式
RS232/RS485：电气传输方式
Modbus RTU：常见上层协议
```

## RS232

RS232 常见于传统串口设备、仪器和工控机。

特点：

- 点对点。
- 使用正负电压。
- 逻辑电平可能与 TTL UART 反相。
- 不能直接接 MCU UART 引脚。

常用转换芯片：

```text
MAX232
MAX3232
SP3232
```

## RS485

RS485 是差分总线，适合长距离和工业现场。

特点：

- A/B 差分线。
- 抗干扰较强。
- 支持多节点总线。
- 常见两线半双工。
- 常搭配 Modbus RTU。

典型结构：

```text
MCU UART <-> RS485 Transceiver <-> A/B Bus
```

## 终端和偏置

长线 RS485 两端通常加：

```text
120Ω 终端电阻
```

空闲状态可能需要偏置电阻，避免总线悬空误判。

## 常见错误

| 现象 | 优先检查 |
| --- | --- |
| TTL UART 直接接 RS232 | 需要 RS232 转换芯片 |
| TTL UART 直接接 RS485 A/B | 需要 RS485 收发器 |
| RS485 最后字节丢失 | DE 方向过早关闭 |
| 多节点不稳定 | 终端、偏置、拓扑、地址冲突 |
| A/B 接了还不通 | 厂商 A/B 命名可能相反 |
