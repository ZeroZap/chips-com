# Communication Debug Playbook

## 目标

本文提供跨协议排障方法，用于把问题快速定位到物理层、电气层、链路层、协议层、驱动层或应用层。

## 第一原则：先分层

任何通信问题先按层拆开：

```text
物理连接：线、连接器、方向、终端、供电
电气质量：电平、差分、上拉、阻抗、时钟、噪声
链路机制：地址、仲裁、枚举、ACK、错误状态
协议格式：帧、长度、CRC、命令、寄存器、对象字典
驱动软件：中断、DMA、缓存、状态机、超时、重试
应用语义：单位、缩放、字节序、状态机、权限
```

不要一开始就改应用代码。

先证明底层链路可靠。

## 通用排障顺序

1. 确认供电和复位。
2. 确认引脚复用和方向。
3. 确认电平域和收发器。
4. 用示波器看物理波形。
5. 用协议分析工具解码。
6. 降低速率验证。
7. 读取固定 ID 或状态寄存器。
8. 再测试完整协议事务。
9. 加入超时、重试和错误统计。
10. 长时间压力测试。

## 工具选择

| 问题类型 | 优先工具 |
| --- | --- |
| 电平、边沿、噪声 | 示波器 |
| UART/I2C/SPI/1-Wire/LIN | 逻辑分析仪 |
| CAN/CANopen | CAN 分析仪 |
| Ethernet/Modbus TCP | Wireshark |
| USB | USB 协议分析仪、系统日志 |
| PCIe | LTSSM 状态、lspci、示波器/协议分析仪 |
| MIPI | SoC 错误计数、示波器、厂商工具 |
| EtherCAT | 主站诊断、WKC、ESI、TwinCAT/IgH/SOEM |

## 常见症状映射

| 现象 | 常见方向 |
| --- | --- |
| 完全无响应 | 电源、复位、接线、地址、角色 |
| 低速正常高速异常 | 信号完整性、上拉、阻抗、采样点 |
| 偶发错误 | 干扰、时序裕量、缓冲区、DMA、任务调度 |
| 固定错误值 | 模式配置、字节序、地址偏移、解析错误 |
| 第一包正常后续失败 | 状态机、CS/STOP、缓存、资源释放 |
| 长时间后失败 | 内存泄漏、计数溢出、热漂移、错误恢复缺失 |
| 只在现场失败 | 接地、浪涌、线缆、EMC、温度、电源质量 |

## 串行字节流协议

适用：

```text
UART
RS485
Modbus RTU
自定义串口协议
```

重点检查：

- 波特率。
- 数据位/校验位/停止位。
- TX/RX 或 A/B。
- 帧边界。
- CRC。
- 超时。
- 半双工方向控制。

## 同步板内总线

适用：

```text
I2C
SPI
I3C
SMBus
PMBus
SDIO
```

重点检查：
- 时钟。
- 地址或片选。
- 上拉或推挽模式。
- 采样边沿。
- 事务边界。
- 电平转换。
- 设备状态机。

## 差分现场总线

适用：

```text
RS485
CAN
LIN，单线但车载环境类似要关注 EMC
```

重点检查：

- 终端电阻。
- 偏置。
- 收发器。
- 参考地或隔离。
- 拓扑。
- 线缆。
- 浪涌/ESD。
- 总线冲突。

## 主机外设总线

适用：

```text
USB
PCIe
SDIO
```

重点检查：

- Host/Device 或 RC/EP 角色。
- 枚举。
- 描述符或配置空间。
- 驱动匹配。
- 中断。
- DMA。
- 电源管理。

## 网络协议

适用：

```text
Ethernet
Modbus TCP
工业以太网
```

重点检查：

- Link。
- MAC 地址。
- IP/Mask/Gateway。
- ARP。
- TCP/UDP 端口。
- 应用层分帧。
- Wireshark 抓包。

## 高速图像显示链路

适用：

```text
MIPI CSI
MIPI DSI
```

重点检查：

- 电源时序。
- MCLK。
- RESET/PWDN。
- I2C 初始化。
- Lane 数和顺序。
- 像素格式。
- 显示时序。
- ISP/Panel 驱动。

## 日志建议

每个协议驱动应记录：

- 初始化成功/失败。
- 地址、ID 或枚举信息。
- 错误码。
- 超时次数。
- CRC 或校验错误次数。
- 总线恢复次数。
- 链路断开次数。
- 状态机当前状态。

现场日志要能区分：

```text
没连上
连上但协议错
协议对但业务状态不允许
```

## 最小复现原则

复杂系统失败时，降级到最小链路：

```text
单主控
单设备
最低速率
最短线缆
最简单命令
读取固定 ID
```

最小链路稳定后，再逐步增加：

- 速度。
- 设备数量。
- 线缆长度。
- DMA。
- 中断。
- 完整业务协议。

## 结论

通信排障不是猜协议，而是按层排除。

先证明：

```text
电气是对的
时序是对的
帧是对的
状态机是对的
应用解释是对的
```

再进入复杂业务逻辑。
