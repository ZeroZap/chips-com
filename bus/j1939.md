# J1939

## 定位

J1939 是基于 CAN 的商用车和工程机械通信协议，常用于柴油发动机、变速箱、仪表和车身控制。

## 核心特点

- 基于 CAN 扩展帧。
- 使用 29-bit ID。
- 定义 PGN 和 SPN。
- 面向商用车和重型设备。
- 支持多包传输。

## PGN 和 SPN

`PGN` 是 Parameter Group Number，表示一组参数。

`SPN` 是 Suspect Parameter Number，表示具体参数。

例如发动机转速、冷却液温度、车辆速度等都可能有对应 SPN。

## 与裸 CAN 的区别

裸 CAN 只定义帧和 ID。

J1939 定义了 ID 中字段含义、参数组、数据解释和网络管理规则。

## 常见场景

- 发动机 ECU。
- 仪表盘。
- 变速箱控制。
- 工程机械控制器。
- 车队监控终端。

## 工程注意

- 使用扩展帧。
- 注意 PGN/SPN 数据缩放和偏移。
- 多包传输需要处理 BAM/RTS/CTS。
- DBC 或协议数据库非常重要。
