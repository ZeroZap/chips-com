# RS485 + Modbus RTU Practical Guide

## 目标

本文用于把 RS485 物理层和 Modbus RTU 协议联合调通，适合工业仪表、传感器、PLC、采集模块和电机控制器。

## 分层关系

```text
Modbus RTU：地址、功能码、寄存器、CRC
UART：波特率、数据位、校验位、停止位
RS485：A/B 差分物理层、半双工、终端电阻
```

不要把 RS485 和 Modbus 混为一谈。

## 硬件接线

```text
MCU TX -> RS485 DI
MCU RX <- RS485 RO
GPIO   -> RS485 DE/RE
A/B    -> RS485 总线
GND    -> 参考地或隔离参考
```

总线建议：

```text
设备1 ---- 设备2 ---- 设备3 ---- 设备4
```

不建议长星型连接。

## 终端和偏置

长线总线两端各加：

```text
120Ω
```

不要每个节点都加终端电阻。

空闲总线可使用 failsafe 偏置，避免无发送时状态漂移。

## 串口参数

常见参数：

```text
9600 8N1
9600 8E1
19200 8N1
115200 8N1
```

主站和所有从站必须一致：

```text
baud rate
data bits
parity
stop bits
slave address
```

## 方向控制

半双工 RS485 发送前：

```text
DE = 1
```

发送完成后：

```text
等待 UART TC 标志
DE = 0
```

不要只等发送缓冲区空，要等移位寄存器真正发完。

## 读保持寄存器示例

请求：

```text
01 03 00 00 00 02 C4 0B
```

含义：

```text
地址 1
功能码 03
从 0x0000 开始读 2 个 holding registers
CRC = C4 0B
```

响应：

```text
01 03 04 00 64 00 C8 BA 7A
```

含义：

```text
返回 4 字节数据
寄存器1 = 100
寄存器2 = 200
```

## 地址换算

手册写：

```text
40001
```

协议里常发：

```text
0x0000
```

手册写 `40010`，协议里可能是 `0x0009`。不同厂商可能从 0 或 1 开始，必须实测确认。

## 调试步骤

1. 用 USB-RS485 工具和上位机先单独读从站。
2. 确认 A/B，必要时交换。
3. 确认串口参数和从站地址。
4. 从功能码 03 读取一个已知寄存器开始。
5. 确认 CRC 低字节在前。
6. MCU 接入后重点检查 DE/RE 方向时序。
7. 多从站时逐个添加，避免地址冲突。

## 常见问题

| 现象 | 优先检查 |
| --- | --- |
| 从站无响应 | A/B、地址、波特率、功能码 |
| 响应乱码 | 校验位、波特率、A/B、终端 |
| 最后字节丢失 | DE 过早关闭 |
| CRC 错 | 字节序、帧长度、串口丢字节 |
| 多设备冲突 | 地址重复、多个主站、广播误用 |

## 实战检查清单

- A/B 和参考地处理正确。
- 两端终端电阻正确。
- UART 参数一致。
- DE/RE 等待 TC 后切回接收。
- Modbus 地址和寄存器偏移确认。
- CRC16 低字节在前。
