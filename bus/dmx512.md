# DMX512

## 定位

DMX512 是灯光控制常用协议，广泛用于舞台灯光、建筑灯光和演出设备控制。

## 核心特点

- 单向控制为主。
- 基于差分物理层，常用 RS485 类收发器。
- 典型一帧包含最多 512 个通道。
- 主控周期性广播通道值。
- 接收设备按起始地址读取对应通道。

## 帧概念

典型 DMX 帧包含：

```text
Break
Mark After Break
Start Code
Channel Data 1..512
```

每个通道通常 8-bit，表示亮度、颜色、位置或其他控制量。

## 常见场景

- 舞台灯。
- 调光器。
- RGB 灯带控制器。
- 烟机、摇头灯等演出设备。

## 与 Modbus RTU 的区别

DMX512 更像周期性广播控制流。

Modbus RTU 是主站请求、从站响应的工业寄存器协议。

二者都可能使用差分物理层，但协议语义完全不同。

## 工程注意

- 方向通常是控制台到灯具。
- 需要正确处理 Break 时序。
- 线缆和终端影响长距离稳定性。
- 灯具地址配置决定读取哪些通道。

## 常见错误

| 现象 | 优先检查 |
| --- | --- |
| 灯不响应 | 地址、接线、Break、收发器方向 |
| 通道错位 | 灯具起始地址、通道数量 |
| 闪烁 | 刷新率、线缆、终端、干扰 |
