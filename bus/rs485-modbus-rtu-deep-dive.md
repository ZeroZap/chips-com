# RS485 + Modbus RTU Deep Dive

## 目标

本文深入 RS485 + Modbus RTU 工程细节，重点解决半双工方向控制、终端/偏置/隔离、防护、帧间隔、CRC、寄存器地址、主站轮询、异常响应、多从站冲突和现场排障。

## 分层模型

先把三层分清：

```text
RS485：差分物理层，解决信号怎么传远
UART：字节帧格式，解决 bit 怎么变成字节
Modbus RTU：应用协议，解决这些字节代表什么
```

典型链路：

```text
MCU UART <-> RS485 Transceiver <-> A/B Bus <-> RS485 Device
```

## RS485 是半双工总线

两线 RS485 常见为半双工：

```text
同一时间只能一个节点发送
其他节点接收
```

如果多个节点同时发送，会出现总线冲突。

Modbus RTU 采用主站请求、从站响应的方式，正好适合半双工 RS485：

```text
主站问
被点名的从站答
其他从站保持安静
```

## DE/RE 方向控制

RS485 收发器常见引脚：

```text
DI：Driver Input，接 MCU TX
RO：Receiver Output，接 MCU RX
DE：Driver Enable，发送使能
/RE：Receiver Enable，接收使能，低有效
A/B：差分总线
```

常见做法：

```text
DE 和 /RE 接同一个 GPIO
GPIO = 1：发送
GPIO = 0：接收
```

关键点：

```text
发送完成后必须等待 UART TC，而不是只等 TXE
```

`TXE` 通常表示发送数据寄存器空。

`TC` 才表示移位寄存器和停止位真正发完。

过早关闭 DE 会导致最后 1 个字节或最后几个 bit 丢失。

## 自动方向控制

有些 RS485 收发器或电路支持自动方向控制。

优点：

- 减少 GPIO。
- 软件更简单。

风险：

- 低波特率下方向保持时间不合适。
- 帧间隔被拉长。
- 特殊收发时序难控制。
- 调试时不容易判断是否方向问题。

工业项目中，GPIO 明确控制 DE/RE 更可控。

## A/B 命名混乱

RS485 厂商对 A/B 命名不完全统一。

可能出现：

```text
A 接 A，B 接 B，结果不通信
```

排查建议：

- 看收发器手册中逻辑 1 时 A/B 谁高。
- 看设备手册端子定义。
- 通信不通时可交换 A/B 验证。
- 用示波器观察差分电压。

## 终端电阻

长线 RS485 主干两端通常各加：

```text
120Ω
```

作用：

```text
匹配线缆阻抗，减少反射
```

注意：

- 只加在总线最远两端。
- 不要每个节点都加。
- 短线低速可能不加也能跑，但不代表规范。
- 多个终端并联会导致负载过重。

## 偏置电阻

总线无人发送时，A/B 可能处于不确定状态。

偏置电阻用于让空闲总线保持已知状态。

常见叫法：

```text
failsafe bias
idle bias
```

现代收发器可能内置 failsafe，但长线和复杂现场仍要结合系统设计确认。

偏置通常只在一处或少数位置配置，不应每个节点都强偏置。

## 隔离和防护

工业现场常见风险：

- 地电位差。
- 浪涌。
- ESD。
- EFT。
- 电机和继电器干扰。
- 长线感应噪声。

常见保护设计：

```text
隔离 RS485 收发器
隔离 DC/DC
TVS
共模电感
保险丝/限流
屏蔽双绞线
合理接地
```

注意：

```text
RS485 是差分，不代表完全不需要参考地或隔离策略。
```

## 拓扑

推荐总线型：

```text
Device1 ---- Device2 ---- Device3 ---- Device4
```

避免长星型：

```text
        Device2
          |
Device1 --+-- Device3
          |
        Device4
```

分支线越短越好。

高速、长线、多节点场景尤其要注意拓扑。

## Modbus RTU 帧边界

Modbus RTU 没有固定起始符和结束符。

它靠静默时间区分帧：

```text
帧间隔 >= 3.5 个字符时间
帧内间隔通常不能超过 1.5 个字符时间
```

字符时间取决于波特率和串口格式。

例如 `9600 8N1` 一字符约 10 bit：

```text
10 / 9600 = 1.0417 ms
3.5 字符约 3.65 ms
```

工程实现中常用：

- UART IDLE 中断。
- 定时器超时。
- DMA + IDLE。
- 接收状态机。

## CRC16

Modbus RTU 使用 CRC16。

注意：

```text
CRC 低字节在前，高字节在后
```

例如帧末尾：

```text
C4 0B
```

表示 CRC 值：

```text
0x0BC4
```

常见错误：

- CRC 算法多项式错。
- 初值错。
- 高低字节顺序反。
- 把 CRC 字节也参与计算。
- 帧长度判断错导致 CRC 错。

## 寄存器地址偏移

Modbus 文档可能写：

```text
40001
40010
```

协议帧里通常发偏移地址：

```text
40001 -> 0x0000
40010 -> 0x0009
```

但不同厂商文档可能从 0 或从 1 开始。

调试方法：

```text
用上位机工具读取相邻地址
对照设备显示值
确认是否需要减 1
```

## 数据字节序

Modbus 单个 16-bit 寄存器通常高字节在前：

```text
00 64 = 100
```

但 32-bit 数据跨两个寄存器时有多种顺序：

```text
AB CD 12 34
CD AB 34 12
12 34 AB CD
34 12 CD AB
```

常见叫法：

- Big-endian word order。
- Little-endian word order。
- Word swap。
- Byte swap。

设备手册必须明确，否则要通过已知值实测。

## 常用功能码工程注意

`03 Read Holding Registers`：

```text
读可读写寄存器，最常用
```

`04 Read Input Registers`：

```text
读只读输入寄存器
```

`06 Write Single Register`：

```text
写单个保持寄存器，响应通常回显请求
```

`10 Write Multiple Registers`：

```text
写多个保持寄存器，注意字节数和寄存器数匹配
```

常见错误：

- 用 03 读了只支持 04 的地址。
- 写只读寄存器。
- 写入值超范围。
- 写多个寄存器时字节数不匹配。

## 异常响应

异常响应功能码：

```text
原功能码 + 0x80
```

例如请求 `03`，异常响应为：

```text
83
```

常见异常码：

| 异常码 | 含义 |
| --- | --- |
| 01 | 非法功能 |
| 02 | 非法数据地址 |
| 03 | 非法数据值 |
| 04 | 从站设备故障 |

不要把异常响应当成 CRC 错或乱码。

异常响应说明设备收到了请求，但拒绝执行。

## 主站轮询策略

主站应避免无节制轮询。

建议：

- 按设备分组轮询。
- 高频数据和低频参数分开。
- 失败设备降频重试。
- 连续超时后标记离线。
- 离线设备低频探测恢复。
- 避免广播读取。

典型状态：

```text
online
timeout
offline
recovering
```

## 超时设置

超时要考虑：

- 波特率。
- 请求长度。
- 响应长度。
- 从站处理时间。
- DE/RE 切换时间。
- 帧间隔。

不要用固定极短超时适配所有波特率。

例如 9600 baud 下读多个寄存器，几十毫秒并不夸张。

## 广播地址 0

地址 `0` 通常是广播。

特点：

```text
从站执行，但不响应
```

适合：

- 同步写某个参数。
- 广播控制。

不适合：

- 读取。
- 需要确认执行结果的操作。

广播误用可能导致多个设备同时动作，必须谨慎。

## 多主问题

标准 Modbus RTU on RS485 通常假设一个主站。

如果多个主站接入同一总线，会造成：

- 请求冲突。
- 从站响应冲突。
- 难以保证帧间隔。
- 随机通信失败。

如确实需要多主，必须有额外仲裁机制或改用更合适总线。

## 从站实现注意

从站设备实现 Modbus RTU 时要注意：

- 地址过滤。
- 广播处理。
- CRC 校验。
- 功能码合法性。
- 寄存器范围检查。
- 异常响应。
- 帧间隔判断。
- 半双工发送方向控制。
- 响应延时不要过长。

从站不应响应其他地址请求，也不应响应广播。

## 主站实现注意

主站实现应包含：

- 请求队列。
- 超时管理。
- 重试策略。
- CRC 校验。
- 异常响应解析。
- 地址和功能码匹配检查。
- 从站状态管理。
- 总线空闲时间控制。
- 统计错误类型。

不要只写一个阻塞式 `send_and_wait` 后到处调用。

## 与上位机工具联调

推荐先用成熟工具确认设备：

- Modbus Poll。
- QModMaster。
- modpoll。
- 厂商调试软件。
- USB-RS485 转换器。

步骤：

1. 上位机工具读通从站。
2. 记录请求和响应帧。
3. MCU 复现相同帧。
4. 对比波形和字节。
5. 再接入完整系统。

## 逻辑分析仪和示波器

逻辑分析仪看：

- UART 解码是否正确。
- 请求和响应字节。
- CRC。
- 帧间隔。
- DE 控制时序。

示波器看：

- A/B 差分电压。
- 终端后波形反射。
- 共模噪声。
- DE 关闭是否截断最后 bit。
- 总线空闲偏置状态。

## 常见故障定位

| 现象 | 优先检查 |
| --- | --- |
| 完全无响应 | A/B、地址、波特率、校验、供电 |
| 偶发超时 | 终端、干扰、轮询过快、从站处理慢 |
| CRC 错 | 干扰、方向截断、波特率、帧边界 |
| 最后字节错 | DE 过早关闭 |
| 返回异常 02 | 寄存器地址偏移或范围错 |
| 返回异常 03 | 写入值非法 |
| 多设备接入后异常 | 地址冲突、终端过多、拓扑错误 |
| 低速正常高速异常 | 线缆、终端、拓扑、收发器速度 |

## 深度检查清单

- RS485 收发器方向控制等待 UART TC。
- A/B 定义已通过手册或实测确认。
- 总线两端终端电阻正确。
- 偏置电阻和 failsafe 设计合理。
- 工业现场有隔离和浪涌/ESD 防护。
- 拓扑为总线型，分支足够短。
- 串口参数和所有从站一致。
- Modbus RTU 帧间隔满足要求。
- CRC16 低字节在前。
- 寄存器地址偏移已实测确认。
- 32-bit 数据字节序已确认。
- 主站有超时、重试、离线和恢复策略。
- 从站正确处理广播和异常响应。
- 上位机工具已验证单设备通信。
