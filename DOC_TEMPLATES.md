# Documentation Templates

## 目标

本文提供知识库新增协议和实战文档时的统一模板，保证后续内容结构一致。

## 协议基础文档模板

```markdown
# 协议名称

## 定位

说明协议解决什么问题，常见在哪些芯片、模块或系统中。

## 核心特点

- 信号线数量。
- 同步或异步。
- 点对点或多设备。
- 全双工或半双工。
- 典型速率和距离。

## 硬件连接

说明引脚、收发器、电平转换、终端电阻、上拉/下拉、时钟和电源要求。

## 通信流程

说明初始化、读写、响应、错误处理和典型事务。

## 常见应用

列出典型芯片、模块和产品场景。

## 常见错误

用表格列出现象和优先检查方向。

## 延伸阅读

链接到 practical、deep-dive、debug 文档。
```

## 实战文档模板

```markdown
# 协议 Practical Guide

## 目标

说明本文解决哪些工程调试问题。

## 最小接线

给出最小硬件连接和注意事项。

## 关键参数

列出必须匹配的参数。

## 典型流程

给出读、写、枚举或初始化流程。

## 调试步骤

按从硬件到协议的顺序列步骤。

## 常见故障定位

| 现象 | 优先检查 |
| --- | --- |
|  |  |

## 实战检查清单

- 关键检查项。
```

## 深入文档模板

```markdown
# 协议 Deep Dive

## 目标

说明深入主题和边界。

## 分层模型

说明物理层、链路层、协议层、应用层关系。

## 关键机制

逐节解释协议最容易出问题的机制。

## 工程设计

说明硬件、驱动、状态机、错误处理、日志和测试。

## 常见故障定位

用表格总结症状和排查方向。

## 深度检查清单

用于评审原理图、驱动和系统测试。
```

## 对比文档模板

```markdown
# A vs B

## 结论先行

用一句话说明各自适合什么。

## 对比表

| 项目 | A | B |
| --- | --- | --- |

## 选型建议

按场景给出选择。

## 常见误区

说明容易混淆的点。
```

## 故障表模板

```markdown
| 现象 | 可能层级 | 优先检查 | 工具 |
| --- | --- | --- | --- |
| 无响应 | 物理/电源 | 电源、复位、接线 | 万用表、示波器 |
| 解码错 | 时序/协议 | 速率、模式、采样点 | 逻辑分析仪 |
```
