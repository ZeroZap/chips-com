# Production Test Strategy

## 目标

本文整理通信接口量产测试策略，确保硬件焊接、接口连通性、基础协议和关键功能可在生产阶段验证。

## 通用原则

- 测试越早发现硬件问题越好。
- 优先测试连接和电平，再测试复杂协议。
- 每个外部接口至少有一个可自动化测试项。
- 测试结果应可记录、可追溯。
- 失败项要能定位到硬件、软件或外设。

## 测试层级

1. 上电和电源测试。
2. 引脚连通性测试。
3. 接口最小通信测试。
4. 目标设备 ID 读取。
5. 功能读写测试。
6. 压力和边界测试。
7. 结果记录和序列号绑定。

## UART

测试项：

- 回环测试。
- 固定字符串收发。
- 波特率配置验证。
- RS485 DE/RE 方向测试。

## I2C

测试项：

- 扫描目标地址。
- 读取 Chip ID。
- 写读可恢复寄存器。
- 总线恢复测试，视产品要求。

## SPI/QSPI

测试项：

- 读取 JEDEC ID。
- 小块读写擦除。
- QSPI 模式读取。
- XIP 基础校验，视产品要求。

## CAN/RS485

测试项：

- 与测试治具通信。
- 固定帧收发。
- 终端电阻检测。
- Modbus/CANopen 基础对象读取。

## USB

测试项：

- 枚举成功。
- VID/PID 正确。
- CDC/HID/MSC 功能测试。
- 插拔重连测试。

## Ethernet

测试项：

- PHY Link。
- 读取 PHY ID。
- Ping。
- TCP/UDP 回环。
- MAC 地址写入和唯一性检查。

## PCIe/SDIO/MIPI

测试项：

- PCIe 枚举和 Link Width/Speed。
- SDIO 设备枚举和固件加载。
- MIPI Sensor ID 或 Panel 初始化。
- 高速接口可通过功能性测试替代完整信号测试。

## 测试记录

建议记录：

```text
产品序列号
硬件版本
固件版本
测试时间
测试工位
测试人员/治具 ID
每项测试结果
失败日志
```

## 结论

量产测试不应追求覆盖所有协议细节，而应覆盖：

```text
焊接正确
电平正确
接口可通信
关键外设存在
基础功能可用
```
